# ПРОСТОЕ ОБЪЯСНЕНИЕ: Как определить MA по графикам

## Что смотреть на графиках MA?

**ВСЕГДА СМОТРИМ НА СРЕДНИЙ СТОЛБЕЦ — ACF!**

## Пошаговая инструкция:

### 1. Открываем графики MA симуляций (нижние 4 ряда)

### 2. Смотрим на СРЕДНИЙ столбец (ACF) в каждом ряду:

**MA(1) — первый ряд:**
```
ACF график:
Lag 1: [■■■] ← большой столбик ВЫХОДИТ за синие линии = ЗНАЧИМЫЙ
Lag 2: [  ] ← внутри синих линий = НЕЗНАЧИМЫЙ
Lag 3: [  ] ← внутри синих линий = НЕЗНАЧИМЫЙ
...все остальные тоже незначимые

ИТОГО: 1 значимый пик → MA(1)
```

**MA(2) — второй ряд:**
```
ACF график:
Lag 1: [■■■] ← значимый
Lag 2: [■■■] ← значимый
Lag 3: [  ] ← незначимый
...все остальные незначимые

ИТОГО: 2 значимых пика → MA(2)
```

**MA(3) — третий ряд:**
```
ACF график:
Lag 1: [■■■] ← значимый
Lag 2: [■■■] ← значимый
Lag 3: [■■■] ← значимый
Lag 4: [  ] ← незначимый
...все остальные незначимые

ИТОГО: 3 значимых пика → MA(3)
```

**MA(4) — четвёртый ряд:**
```
ACF график:
Lag 1: [■■■] ← значимый
Lag 2: [■■■] ← значимый
Lag 3: [■■■] ← значимый
Lag 4: [■■■] ← значимый
Lag 5: [  ] ← незначимый
...все остальные незначимые

ИТОГО: 4 значимых пика → MA(4)
```

### 3. Сравниваем с реальными данными

**Наш ACF для diff(log_jj):**
- Lag 1: -0.5068 ← **БОЛЬШОЙ ПИК!** Выходит за синие линии
- Lag 2: меньше
- Lag 3: меньше

**Это похоже на MA(1)!**
- Один большой пик на лаге 1
- Затем затухание

### 4. Правило выбора:

**Считаем значимые пики (выходят за синие линии) ДО первого обрыва:**

- 1 пик → MA(1) ✅ **НАШ СЛУЧАЙ**
- 2 пика → MA(2)
- 3 пика → MA(3)
- 4 пика → MA(4)

## Визуальная подсказка:

```
Значимый пик (выходит за синие линии):
     │

━━━━━━━━━━━━━ синие линии

Незначимый (внутри синих линий):
     │

━━━━━━━━━━━━━ синие линии
█████████████
```

## Почему MA(1) для наших данных? ⚠️ ВАЖНО!

### Вопрос: А почему не MA(4), если есть пики на лагах 1, 3, 4?

**Хороший вопрос!** В реальных данных ACF имеет МНОГО значимых пиков:
- Lag 1: -0.5068 ← САМЫЙ СИЛЬНЫЙ
- Lag 3: -0.4028
- Lag 4: 0.7314 ← СЕЗОННОСТЬ!
- Lag 5, 7, 8, 9, 11, 12...

### Правило: Как определить MA в смешанном процессе

**1. Первый лаг — главный индикатор MA:**

Смотрим ТОЛЬКО на лаг 1:
- Если есть сильный пик на лаге 1 → это MA(1)
- Это "чистый" эффект от прошлых ошибок

**2. Остальные пики могут быть:**
- **Сезонность**: лаги 4, 8, 12 (кратные 4) — это НЕ MA!
- **AR эффекты**: постепенное затухание
- **Смешанные эффекты**: комбинация AR+MA+сезонность

**3. Почему не MA(4)?**

Если бы это был ЧИСТЫЙ MA(4) (как в симуляциях):
```
MA(4) должен иметь:
Lag 1: ███ (сильный)
Lag 2: ███ (сильный)
Lag 3: ███ (сильный)
Lag 4: ███ (сильный)
Lag 5: [всё внутри синих линий] ← РЕЗКИЙ ОБРЫВ
```

А у нас:
```
Lag 1: █████ (очень сильный -0.51) ← MA(1)
Lag 2: █ (слабый)
Lag 3: ███ (средний, но это может быть AR)
Lag 4: ████ (сильный, но это СЕЗОННОСТЬ!)
```

**Вывод:**
- Lag 1 сильный и стоит особняком → MA(1)
- Лаги 4, 8, 12 — это сезонность, не MA
- Поэтому ARIMA(4,1,1) имеет смысл: AR(4) ловит зависимости на лагах 1-4, MA(1) ловит эффект на лаге 1
